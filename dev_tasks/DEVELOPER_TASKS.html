🚀 Этапы запуска
🤖
Этап 1 — AI Генератор
Anthropic Claude API · /generator страница оживает
⏱ 10-15 минут
  • Зарегистрироваться на console.anthropic.com (только email, без телефона)
  • Создать API ключ: Settings → API Keys → Create Key
  • Пополнить баланс: минимум $5 (хватит на ~2000 запросов)
  • Открыть .env.local и заменить ANTHROPIC_API_KEY=your_anthropic_api_key_here на реальный ключ
  • Перезапустить сервер: npm run dev
  • Проверить: зайти на /generator и нажать «Генерировать»
Результат: Страница /generator генерирует реальные сценарии через Claude API. Модель: claude-sonnet-4-6, до 1500 токенов, поддержка: сценарий / хук / CTA / заголовок / описание / идеи.
🗄️
Этап 2 — База данных
Supabase PostgreSQL · история контента · лиды
⏱ 30-40 минут
  • Зарегистрироваться на supabase.com (бесплатный план: 500MB, 50k запросов/мес)
  • Создать новый проект: Organization → New Project → выбрать регион EU West
  • Скопировать 3 ключа из Project Settings → API в .env.local
  • В SQL Editor выполнить миграции (таблицы ниже)
  • Включить Row Level Security (RLS) для production

Таблицы для создания:

generated_content

id (uuid, pk) type (text) topic (text) platform (text) tone (text) content (text) created_at (timestamptz)

leads

id (uuid, pk) name (text) phone (text) city (text) source (text) status (text) notes (text) created_at (timestamptz)

scheduled_posts

id (uuid, pk) content_id (uuid, fk) platform (text) scheduled_at (timestamptz) status (text) published_at (timestamptz)
Результат: История сгенерированного контента сохраняется. Лиды из форм записываются в базу. Данные доступны в Supabase Dashboard.
🌐
Этап 3 — Постоянный деплой
GitHub + Vercel · постоянный URL · CI/CD
⏱ 30-45 минут
  • Создать репозиторий на GitHub: github.com → New Repository → adonis-platform (private)
  • В папке проекта: git init → git add . → git commit -m "init" → git push
  • Зарегистрироваться на vercel.com (через GitHub аккаунт — телефон не нужен)
  • New Project → Import GitHub Repository → выбрать adonis-platform
  • В настройках Vercel добавить все переменные из .env.local (Environment Variables)
  • Deploy → получить постоянный URL вида adonis-platform.vercel.app
  • Настроить кастомный домен если есть (Settings → Domains)
Результат: Постоянная ссылка для заказчика. При каждом git push — автоматический деплой за 1-2 минуты.
📨
Этап 4 — Telegram автопостинг
Telegram Bot API · первый реальный постинг · бесплатно
⏱ 2-3 дня
  • Создать бота: написать @BotFather в Telegram → /newbot → получить BOT_TOKEN
  • Добавить бота в канал как администратора с правом публикации
  • Получить ID канала: переслать сообщение из канала боту @getidsbot
  • Добавить в .env.local: TELEGRAM_BOT_TOKEN и TELEGRAM_CHANNEL_ID
  • Создать новый API роут: app/api/telegram/publish/route.ts
  • В роуте использовать Telegram Bot API: POST https://api.telegram.org/bot{TOKEN}/sendMessage
  • Подключить кнопку «Запостить в Telegram» на странице /autopost к новому роуту
  • Для видео: использовать sendVideo метод (поддерживает URL)
  • Для расписания: использовать Vercel Cron Jobs (vercel.json → "crons")
Результат: Реальная публикация текста, изображений и видео в Telegram канал по расписанию. Telegram Bot API бесплатный, без ограничений модерации.
📱
Этап 5 — Instagram + TikTok
Instagram Graph API · Content Publishing API · модерация
⏱ 1-2 недели
  • Instagram: Нужен Facebook Business аккаунт + Instagram Professional аккаунт
  • Создать приложение на developers.facebook.com → добавить продукт Instagram Graph API
  • Получить долгосрочный access_token (60 дней, можно обновлять)
  • Endpoint для публикации: POST /{ig-user-id}/media → POST /{ig-user-id}/media_publish
  • Ограничение: не более 25 публикаций в 24 часа на аккаунт
  • TikTok: Зарегистрировать аккаунт разработчика на developers.tiktok.com
  • Подать заявку на Content Posting API (рассматривают 1-5 рабочих дней)
  • Использовать Video Upload API для загрузки видео
Результат: Публикация Reels и постов в Instagram, публикация видео в TikTok через единый интерфейс /autopost.
🎬
Этап 6 — Генерация видео
HeyGen / RunwayML · реальная Видео-фабрика
⏱ 2-4 недели
  • Вариант A — HeyGen: heygen.com → API Access → аватар + озвучка → $30/мес
  • Endpoint: POST /v1/video.generate с текстом скрипта → получить URL видео
  • Подключить к странице /video-factory: кнопка «Создать видео» → реальный запрос
  • Вариант B — RunwayML: runwayml.com → Gen-3 Alpha API → генерация видео из промпта
  • Вариант C (эконом) — ElevenLabs + D-ID: ElevenLabs для голоса + D-ID для аватара
  • Сохранять статус генерации в Supabase (polling каждые 5 сек пока не готово)
  • Финальное видео сохранять в Supabase Storage или Cloudflare R2
Результат: Полный цикл: сгенерировал сценарий → создал видео → опубликовал в TikTok/Instagram — всё в одном интерфейсе.

💾 SQL миграция для Supabase

Скопировать в Supabase → SQL Editor → New Query → Run

-- Таблица для сгенерированного контента CREATE TABLE generated_content ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, type text NOT NULL, topic text, platform text, tone text, content text NOT NULL, created_at timestamptz DEFAULT now() ); -- Таблица лидов CREATE TABLE leads ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, name text NOT NULL, phone text, city text, source text, status text DEFAULT 'new', notes text, created_at timestamptz DEFAULT now() ); -- Таблица запланированных публикаций CREATE TABLE scheduled_posts ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, content_id uuid REFERENCES generated_content(id), platform text NOT NULL, scheduled_at timestamptz NOT NULL, status text DEFAULT 'pending', published_at timestamptz );
